What We Need

A SAML Enterprise Application in Azure AD (Entra ID) so the OneForma marketing team can sign into OneTake using their existing Microsoft credentials (@centific.com / @oneforma.com accounts).

Our application uses Clerk (clerk.com) as our authentication provider. Clerk acts as the SAML Service Provider (SP) and needs to be connected to Centific's Azure AD as the Identity Provider (IdP).

Setup Steps for Azure AD Admin

Step 1: Create Enterprise Application

  1. Go to Azure Portal → Microsoft Entra ID → Enterprise Applications
  2. Click New Application → Create your own application
  3. Name: OneTake - Clerk SSO
  4. Select: Integrate any other application you don't find in the gallery (Non-gallery)
  5. Click Create

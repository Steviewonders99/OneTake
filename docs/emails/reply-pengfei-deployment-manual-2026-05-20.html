3. Deployment Steps

# Step 1: Pull the image (already in novacert.azurecr.io)
docker pull novacert.azurecr.io/onetake-worker:latest

# Step 2: Run the proxy (standalone test)
docker run -d \
  -p 8080:8080 \
  -e DATABASE_URL="postgresql://sqladm:***@onetake-pg-west01..." \
  -e PROXY_SECRET="secret-will-share-securely" \
  -e PROXY_PORT="8080" \
  novacert.azurecr.io/onetake-worker:latest \
  python db_proxy.py

# Step 3: Verify health
curl http://localhost:8080/health
# Expected: {"ok": true, "db": true}

# Step 4: Test with auth
curl -H "Authorization: Bearer YOUR_SECRET" \
  http://localhost:8080/projects
# Expected: JSON array of 48 projects

# Step 5: Test without auth (should be rejected)
curl http://localhost:8080/projects
# Expected: {"error": "Unauthorized"} (401)

5. Verification Checklist

After deploying, run these checks:
  1. GET /health → {"ok": true, "db": true}
  2. GET /projects (with auth) → 48 projects returned
  3. GET /projects (no auth) → 401 Unauthorized
  4. GET /projects/{any-uuid}/funnel?view=weekly → weekly data
  5. Response headers include: X-Content-Type-Options: nosniff, X-Frame-Options: DENY, Server: onetake-proxy
  6. Rate limiting: 120+ rapid requests → 429 response

6. Security Summary

ControlDetail
AuthenticationBearer token (256-bit) on every request except /health
Token comparisonConstant-time (hmac.compare_digest) — timing-safe
Rate limiting120 requests/minute per IP
Database accessRead-only (no DDL, no writes except materialized view refresh)
Error handlingGeneric errors — never leaks internal details
TransportHTTPS (TLS terminated by ingress/gateway)
Server headeronetake-proxy (hides Python/aiohttp version)

7. For K8s Deployment

If deploying to K8s instead of Container Apps:

# Deployment spec (minimal)
apiVersion: apps/v1
kind: Deployment
metadata:
  name: onetake-proxy
spec:
  replicas: 1
  selector:
    matchLabels:
      app: onetake-proxy
  template:
    metadata:
      labels:
        app: onetake-proxy
    spec:
      containers:
      - name: proxy
        image: novacert.azurecr.io/onetake-worker:latest
        command: ["python", "db_proxy.py"]
        ports:
        - containerPort: 8080
        env:
        - name: DATABASE_URL
          valueFrom:
            secretKeyRef:
              name: onetake-secrets
              key: database-url
        - name: PROXY_SECRET
          valueFrom:
            secretKeyRef:
              name: onetake-secrets
              key: proxy-secret
        - name: PROXY_PORT
          value: "8080"
        readinessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 5
          periodSeconds: 10
        livenessProbe:
          httpGet:
            path: /health
            port: 8080
          initialDelaySeconds: 10
          periodSeconds: 30
        resources:
          requests:
            cpu: 100m
            memory: 128Mi
          limits:
            cpu: 500m
            memory: 256Mi
---
apiVersion: v1
kind: Service
metadata:
  name: onetake-proxy
spec:
  selector:
    app: onetake-proxy
  ports:
  - port: 8080
    targetPort: 8080
